<product_requirements>
The goal is to build StickersXTon, a Telegram Mini App with a mobile-first, dark glassmorphism UI, and a separate web-based Admin Dashboard. Both modules share a FastAPI/Python backend, with MongoDB as the database and React for the frontend, styled with Tailwind CSS and Shadcn UI components. TonConnect SDK integrates wallet functionality.

**User Mini App Features:**
1.  **Bottom Navigation:** Marketplace, Activity, Hot, Profile (Wallet/Settings). Admin menu is hidden for normal users.
2.  **Marketplace:** Displays sticker packs/single stickers as glassmorphism cards (preview, title, number, price (TON/Stars), rarity tag, Buy button). Filters: All / My Stickers / External. Featured carousel. Upcoming collections with countdown and ADD WATCHLIST (redirects to Telegram channel, enforces subscription). Responsive layout.
3.  **Activity Feed:** Categories (All/Paid/Free/Finished). Entries show pack name, action (bought/opened/burned), price. Auto-refresh and simulated events supported. User IDs/wallet addresses are never displayed publicly.
4.  **Hot:** Top 5 collections, configurable by admin (manual/auto).
5.  **Profile:** TonConnect wallet connection, balances (TON, Stars, SXTON points), referrals count, History (Listed/Unlisted), Wallet (connect), Withdraw/Deposit (placeholders), Language selector.
6.  **Roulette/Spin:** Horizontal pack-opening animation, configurable TON payment per spin (mock). Reveals with confetti, Keep/Sell buttons. Admin sets price, pool, probabilities.
7.  **Buy/Sell Flows:** TON via TonConnect; Stars via manual instruction. Min sale price = 1 TON. Auto 2.5% commission. Max 10 listings per collection.
8.  **SXTON Points:** Earned on purchases (500 per sticker, editable) and referrals. Usable for special collections.
9.  **Giveaways/Auctions:** Users create giveaways (paid/free). Admin sets entry requirements. Finished tab shows completed giveaways.
10. **Sticker Burn System:** Admin enables burning for packs. Users burn owned stickers (confirm dialog), removing from circulation and awarding SXTON points (configurable). Burn events appear in Activity feed.
11. **External Stickers Integration:** Fetch metadata via public APIs (mock data initially). External stickers appear in Marketplace filters, not featured home. Admin enables/disables sources, sets per-source commission.
12. **Activity Simulation & Admin Preview:** Admin generates fake activity (configurable amount/day), adjusts min/max, previews. Simulates daily volume.
13. **Referral System (2-level):** Admin configurable (50% L1, 10% L2). Telegram deep links. Rewards in SXTON points.
14. **Multilanguage:** Language selector, text keys in JSON locale files, UI switches without reload.

**Admin Dashboard Features:**
1.  **Authentication:** TON wallet verification + fallback username/password (/). Hidden from normal users.
2.  **Controls:** Manage stickers/packs (CRUD), set price/type (TON/Stars/SXTON), upload media, set number/visibility. Commission settings (global/per-source), No-Commission mode. Enable/disable burn, set burn rewards. Manage Roulette pools/weights. Manage Hot list (manual/auto). External sources toggle + refresh. Analytics dashboard (sales volume, active users, spins, burns, referrals; Real/Simulated data toggle). Manual balance adjustment & export logs.
3.  **Analytics Dashboard Enhancements (Current State):** Displays Total Users, Online Users (live/editable), Total Trading Volume (real/editable), Total Packs, Transactions. **SXTON Token Overview** (Total Distributed, Spent, Circulating).
4.  **SXTON Token Management (Current State):** Panel for Distribution Rules (Points per Purchase, Points per Burn), Token Economy (Enable Spending, Min Token Balance), Transaction History.
5.  **Pack Management (Current State):** Admin can configure allowed purchase types (TON, STARS, SXTON) and require channel subscription (with Channel ID and Link fields).
6.  **Banner Ads Management (Current State):** Admin can create and manage banner ads with links and optional cover images.
7.  **Latest Marketplace Activity (Pending):** Show recent actions (collection, type, amount, pseudonym, timestamp) with filters (collection, action type, time). Admin manual editing.

**General UI/UX:** Dark glassmorphism cards, neon accent, smooth micro-interactions, responsive mobile layout, bottom fixed nav, large tap targets. Animated pack opening, confetti/firework on win, micro-animations for buttons. Loading skeletons, lazy image loading. Cosmic theme with subtle/brighter stars/particles. Accent colors: bright turquoise/green (actions), red (warnings), yellow/green (statuses). Harmonized typography, spacing, visual hierarchy.
