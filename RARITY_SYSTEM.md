# 🎨 Система Рідкості Стікерів

## Як це працює

Система дозволяє вам **самостійно** визначати рідкість кожного стікера за назвою файлу!

### 📝 Формат назв файлів

Використовуйте цей формат для автоматичного розпізнавання рідкості:

```
#<номер>_<рідкість>.<розширення>
```

### ✅ Приклади правильних назв

```
#1_legendary.png      → Legendary (рідкісний)
#2_legendary.jpg      → Legendary
#3_epic.png          → Epic (епічний)
#4_epic.jpg          → Epic
#5_rare.png          → Rare (рідкий)
#6_uncommon.png      → Uncommon (нечастий)
#7_common.png        → Common (звичайний)
#8.png               → Автоматично розраховується (Common для цього паку)
```

### 🎯 Розпізнавальні слова для рідкості

Система розпізнає ці слова (без урахування регістру):

- `legendary` → **Legendary** 🌟
- `epic` → **Epic** ⭐
- `rare` → **Rare** 💎
- `uncommon` → **Uncommon** 🔮
- `common` → **Common** 🌀

### 📊 Розподіл рідкості

**Якщо рідкість НЕ вказана в назві** - система автоматично розраховує:

Для паку з 2500 стікерів:
```
Legendary: 1% → 25 стікерів (#1-25)
Epic:      4% → 100 стікерів (#26-125)
Rare:      10% → 250 стікерів (#126-375)
Uncommon:  25% → 625 стікерів (#376-1000)
Common:    60% → 1500 стікерів (#1001-2500)
```

### 🛠️ Як створити пак з користувацькою рідкістю

1. **Підготуйте фото** за форматом:
   ```
   #1_legendary.png
   #2_epic.png
   #3_epic.png
   #4_rare.png
   #5_rare.png
   #6_uncommon.png
   ...
   #2500.png  (обов'язково починайте з #1!)
   ```

2. **Перейдіть в админ-панель** → Pack Management

3. **Завантажте фото** у форматі:
   - **Upload Mode**: вибрати фото по одному або масово
   - **Або вказати URL**: якщо фото вже в мережі

4. **Система автоматично**:
   - ✅ Прочитає рідкість з назв файлів
   - ✅創користувач непокідені ще вибере - використовуватиме автоматичну формулу
   - ✅ Збереже рідкість кожного стікера в базі

### 💡 Приклади реальних назв

**Лотерея**:
```
#1_legendary.png   ← Головний приз!
#2_epic.png
...
#500_common.png
```

**限量版**:
```
#1_legendary.png
#2_legendary.png
#3_legendary.png
#4_epic.png
...
```

**Всі однакові рідкості** (автоматичний розрахунок):
```
#1.png   → Legendary (автоматично)
#2.png   → Legendary або Epic (за формулою)
#3.png   → Epic або Rare
...
```

### 🎮 При покупці стікерів

Коли користувач купує пак:

1. ✅ Система вибирає **рандомні стікери**
2. ✅ Показує рідкість кожного (з вашої настройки!)
3. ✅ Зберігає у профіль користувача

**Результат**:
```
Купив пак з 10 стікерів, отримав:
- #1 → Legendary ⭐⭐⭐⭐⭐
- #3 → Epic ⭐⭐⭐⭐
- #5 → Rare ⭐⭐⭐
- #7 → Common ⭐
```

### ⚠️ Важливо!

1. **Номери мають починатися з #1** і бути послідовними (#1, #2, #3...)
2. **Розширення не важливо**: `.png`, `.jpg`, `.jpeg`, `.gif` - все працює
3. **Можна змішувати**: деякі з рідкістю, деякі без
   - Вказані з рідкістю → користуються вашою рідкістю
   - Без рідкості → обчислюються автоматично
4. **Регістр не важливо**: `_LEGENDARY`, `_Legendary`, `_legendary` - все одне й те саме

### 🚀 Приклад для художника

Ви можете надати художникам цей формат:

```
Я хочу пак з 100 стікерів:
- #1-5: Legendary (найрідкісні)
- #6-20: Epic
- #21-50: Rare
- #51-100: Common

Назвіть файли так:
#1_legendary.png
#2_legendary.png
...
#5_legendary.png
#6_epic.png
...
#50_rare.png
#51_common.png
...
#100_common.png
```

### 📈 Статистика

При створенні паку ви побачите:
```

Розпізнане розподілення:
- Legendary: 5 (5.0%)
- Epic: 15 (15.0%)
- Rare: 30 (30.0%)
- Common: 50 (50.0%)
```
