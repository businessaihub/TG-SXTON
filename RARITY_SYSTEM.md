# 🎨 Система Рідкості Стікерів

## 🎯 Два рівні рідкості

### 1️⃣ **EDITION** (Категорія паку) - для маркетингу
Це поле при створенні паку для **категоризації** вашої колекції:
- `common` - Звичайний пак 📦
- `limited_edition` - Обмежена серія 🔒
- `legendary_edition` - Легендарна серія 👑
- `seasonal` - Сезонний реліз 🎄
- `exclusive` - Ексклюзивний реліз ⭐

**Призначення**: для фільтрації в каталозі, маркетингу, гарячих колекцій

### 2️⃣ **STICKER RARITY** (Рідкість окремого стікера) - визначається за назвою файлу
Це **справжня рідкість** кожного стікера у паку:
- `Legendary` - рідкісний ⭐⭐⭐⭐⭐
- `Epic` - епічний ⭐⭐⭐⭐
- `Rare` - рідкий ⭐⭐⭐
- `Uncommon` - нечастий ⭐⭐
- `Common` - звичайний ⭐

---

## 📝 Формат назв файлів для рідкості стікерів

Використовуйте цей формат для автоматичного розпізнавання рідкості:

```
#<номер>_<рідкість>.<розширення>
```

### ✅ Приклади правильних назв

```
#1_legendary.png      → Legendary (рідкісний)
#2_legendary.jpg      → Legendary
#3_epic.png          → Epic (епічний)
#4_epic.jpg          → Epic
#5_rare.png          → Rare (рідкий)
#6_uncommon.png      → Uncommon (нечастий)
#7_common.png        → Common (звичайний)
#8.png               → Автоматично розраховується (Common для цього паку)
```

### 🎯 Розпізнавальні слова для рідкості

Система розпізнає ці слова (без урахування регістру):

- `legendary` → **Legendary** 🌟
- `epic` → **Epic** ⭐
- `rare` → **Rare** 💎
- `uncommon` → **Uncommon** 🔮
- `common` → **Common** 🌀

### 📊 Розподіл рідкості при відсутності зазначення

**Якщо рідкість НЕ вказана в назві** - система автоматично розраховує за формулою:

Для паку з 2500 стікерів:
```
Legendary: 1% → 25 стікерів (#1-25)
Epic:      4% → 100 стікерів (#26-125)
Rare:      10% → 250 стікерів (#126-375)
Uncommon:  25% → 625 стікерів (#376-1000)
Common:    60% → 1500 стікерів (#1001-2500)
```

## 🛠️ Як створити пак з користувацькою рідкістю

### Сценарій: Limited Edition пак з 100 стікерів

1. **Підготуйте фото** за форматом:
   ```
   #1_legendary.png      ← Гарячий приз!
   #2_legendary.png
   #3_legendary.png
   #4_legendary.png
   #5_legendary.png      ← 5 легендарних
   
   #6_epic.png
   #7_epic.png
   ...
   #20_epic.png          ← 15 епічних
   
   #21_rare.png
   #22_rare.png
   ...
   #50_rare.png          ← 30 рідких
   
   #51_uncommon.png
   #52_uncommon.png
   ...
   #100_uncommon.png     ← 50 нечастих
   ```

2. **Перейдіть в админ-панель** → Pack Management

3. **Новий пак**:
   - Назва: "Limited Stickers #1"
   - Опис: "Обмежена серія - тільки 100 стікерів"
   - **Edition**: "Limited Edition" 🔒
   - Завантажте всі 100 фото

4. **Система автоматично**:
   - ✅ Прочитає рідкість з назв файлів
   - ✅ Збереже рідкість кожного стікера в базі
   - ✅ Позначить пак як "Limited Edition" для каталогу

5. **Результат**: Коли юзер купить пак, отримає **рандомні стікери з вашою рідкістю**!

## 🎯 Рекомендації для художників

Якщо працюєте з художниками, дайте їм цей чеклист:

```
🎨 ІНСТРУКЦІЯ ДЛЯ ХУДОЖНИКА

Пак: "Space Explorers"
Кількість стікерів: 50

РОЗПОДІЛ РІДКОСТІ:
- #1-2: Legendary (найкруче)
- #3-7: Epic (дуже круто)
- #8-15: Rare (круто)
- #16-50: Common (звичайні)

НАЗВІТЬ ФАЙЛИ:
#1_legendary.png
#2_legendary.png
#3_epic.png
#4_epic.png
#5_epic.png
#6_epic.png
#7_epic.png
#8_rare.png
... і так далі до ...
#50_common.png

ВАЖЛИВО:
✅ Починайте з #1
✅ Номери послідовні (#1, #2, #3...)
✅ Видалення не важливе (.png, .jpg, .jpeg)
✅ Пиши рідкість малими буквами: legendary, epic, rare, uncommon, common
```

---

## ⚠️ Важливо!

1. **Номери мають починатися з #1** і бути послідовними
2. **Розширення не важливо**: `.png`, `.jpg`, `.jpeg`, `.gif` - все працює
3. **Можна змішувати**: деякі з рідкістю, деякі без
   - Вказані з рідкістю → користуються вашою рідкістю
   - Без рідкості → обчислюються автоматично
4. **Регістр не важливо**: `_LEGENDARY`, `_Legendary`, `_legendary` - все одне й те саме
